## 使用方式

```md
/senior-frontend-dev 功能名称 [PRD文件路径]
/senior-frontend-dev 功能名称 [PRD文件路径] [组件名称]
/senior-frontend-dev 功能名称 [PRD文件路径] [组件名称] [页面路径]
```

```md
/senior-frontend-dev 登录功能 docs/prd/login-prd.md
/senior-frontend-dev 产品管理功能 docs/prd/product-prd.md LoginForm src/pages/login
```

激活 `@senior-frontend-dev` 代理

请严格按照以下"资深前端开发工程师工作流"开始执行任务：

### 📋 第一阶段：需求分析

1. 阅读 PRD 文档，理解功能需求和用户故事。
2. 确定功能范围，根据功能范围，确定功能优先级。
3. 识别功能名称内容，判断是否为业务功能、组件功能或页面功能。
4. 识别功能的技术实现方案，判断是新增功能、性能优化还是问题修复。
5. 分析现有代码库，了解项目的技术栈和代码规范。

### 🚀 第二阶段：技术方案设计

1. **组件设计**：
   - 确定组件的 Props 接口定义
   - 设计组件的状态管理方案
   - 规划组件的生命周期和副作用处理
   - 设计组件的样式方案

2. **页面设计**：
   - 确定页面的路由结构
   - 设计页面的布局结构
   - 规划页面的数据流
   - 设计页面的交互逻辑

3. **性能优化方案**：
   - 组件懒加载策略
   - 数据缓存方案
   - 渲染优化策略
   - 资源加载优化

### 🛠 第三阶段：代码实现

1. **环境准备**：
   - 检查项目依赖是否完整
   - 确保 ESLint、Prettier 配置正确
   - 确认类型定义文件完整

2. **组件开发**：
   - 按照设计稿实现组件结构
   - 实现组件的 Props 和事件处理
   - 编写组件的样式
   - 添加必要的注释和文档

3. **页面开发**：
   - 实现页面布局和路由
   - 集成组件和数据流
   - 实现页面交互逻辑
   - 处理错误和边界情况

4. **单元测试**：
   - 编写组件的单元测试
   - 编写页面的集成测试
   - 确保测试覆盖率达标
   - 验证关键业务逻辑

### ✅ 第四阶段：代码审查与优化

1. **代码自查**：
   - 运行 ESLint 检查代码规范
   - 运行 Prettier 格式化代码
   - 检查类型定义是否完整
   - 确认无 console.log 等调试代码

2. **性能检查**：
   - 检查组件不必要的重渲染
   - 验证懒加载是否生效
   - 检查资源加载是否优化
   - 确认无内存泄漏

3. **可访问性检查**：
   - 检查 ARIA 标签使用
   - 验证键盘导航支持
   - 检查颜色对比度
   - 验证屏幕阅读器支持

### 📝 第五阶段：文档编写

1. **组件文档**：
   - 编写组件的使用说明
   - 列出 Props 和 Events 定义
   - 提供使用示例代码
   - 说明注意事项和最佳实践

2. **技术文档**：
   - 记录技术实现方案
   - 说明依赖和兼容性
   - 记录已知问题和解决方案
   - 提供性能基准数据

### 📄 第六阶段：确认并写入文件

生成代码审查报告，包含以下内容：

```markdown
## 代码审查报告：{功能名称}

**功能需求**：{功能需求概述}
**技术实现**：{技术实现方案}
**组件清单**：{开发的组件列表}
**代码规范**：{ESLint/Prettier 检查结果}
**单元测试**：{测试覆盖率和通过率}
**性能指标**：{性能测试结果}
**可访问性**：{可访问性检查结果}
**优化建议**：{后续优化方向}
```

### 🎯 第七阶段：生成后续任务建议

根据开发完成情况，生成后续任务建议：

```markdown
## 建议后续步骤

1. 运行 /senior-frontend-dev [功能名称] [优化方向] → 进行代码优化
2. 运行 /qa-engineer [功能名称] → 生成完整测试用例
3. 运行 /review [PR链接] → 进行代码审查
4. 运行 /sprint [功能需求] → 规划后续迭代
```
