## 使用方式

```md
/fed-arch 功能名称 [PRD文件路径]
/fed-arch 功能名称 [PRD文件路径] [组件名称]
/fed-arch 功能名称 [PRD文件路径] [组件名称] [组件库名称]
/fed-arch 功能名称 [PRD文件路径] [组件名称] [组件库名称] [组件库文档路径]
```

```md
/fed-arch 登录功能 docs/prd/login-prd.md
/fed-arch 产品管理功能 docs/prd/product-prd.md
```

激活 `@frontend-architect` 代理

请严格按照以下“前端架构师工作流”开始执行任务：

### 📋 第一阶段：需求分析

1. 阅读 PRD 文档，理解功能需求和用户故事。
2. 确定功能范围，根据功能范围，确定功能优先级。
3. 识别功能名称内容，判断是否为业务功能。
4. 识别功能名称内容，判断是否为组件功能。
5. 识别功能名称内容，判断是否为组件库功能。
6. 识别功能名称内容，判断是添加新功能，还是对当前功能进行性能优化、问题排查

### 🚀 第二阶段：前端架构设计

1. 确定前端技术栈，并生成技术选型报告。
2. 确定功能架构，并生成架构图。
3. 确定前端目录结构，并生成目录结构图。
4. 确定前端组件结构，并生成组件结构图。
5. 确定前端数据流，并生成数据流图。
6. 确定前端样式规范，并生成样式规范文档。
7. 确定前端代码规范，并生成代码规范文档。
8. 确定前端组件库，并生成组件库文档。
9. 确定前端性能优化方案，并生成性能优化报告。
10. 确定前端问题排查方案，并生成问题排查报告。

### 🛠 第三阶段：前端架构搭建

1. 初始化项目，并生成项目骨架。
2. 搭建前端脚手架，并生成脚手架文档。
3. 搭建前端组件库，并生成组件库文档。
4. 搭建前端数据流，并生成数据流文档。
5. 搭建前端样式规范，并生成样式规范文档。
6. 搭建前端代码规范，并生成代码规范文档。
7. 搭建前端性能优化方案，并生成性能优化文档。
8. 搭建前端问题排查方案，并生成问题排查文档

## 第四阶段：确认并写入文件

```
docs/arch/{功能英文名}-fed-arch.md
```

内容包含以下内容：

```
## 功能架构设计：{功能名称}

**功能需求**：{功能需求}
**技术选型**：{技术选型}
**功能架构**：{功能架构}
**目录结构**：{目录结构}
**组件结构**：{组件结构}
**数据流**：{数据流}
**样式规范**：{样式规范}
**代码规范**：{代码规范}
**组件库**：{组件库}
**性能优化**：{性能优化}
**问题排查**：{问题排查}
```

## 第五阶段：生成后续任务建议

根据前端架构设计，生成后续的前端开发任务建议，内容包含：

```
## 建议后续步骤

1. 运行 /ui-spec {功能名称} → 搭建前端组件库
2. 运行 /api-design {功能名称} → 搭建前端数据流
3. 运行 /test-plan {功能名称} → 搭建前端样式规范
4. 运行 /code-spec {功能名称} → 搭建前端代码规范
5. 运行 /sprint {功能需求} → 搭建前端性能优化方案
```
